# Markdown Style Guide

## Best Practices
- Keep documentation inside the docs/ directory
- Use relative paths for local links
- Use plugin links for interactive workflows
- URL‑encode Python code in plugin links
- Use headings to structure content
- Prefer fenced code blocks for commands
- Keep documents short and focused
- Use tables for structured information
- Use breadcrumbs to help users understand navigation context

## How-To link Markup docs

When writing markup documentation, use the following format for
the link to pass the proper location to the markup link plugin.

plugin://markdown_docs?file_path=docs/style_guide.md

## Link Selector Popup
Press L to open the link selector popup.

- Arrow keys navigate
- Enter selects
- Mouse click selects
- Popup is centered and non‑fullscreen

Use clear link labels to make selection intuitive.
